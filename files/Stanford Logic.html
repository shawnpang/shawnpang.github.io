    • Reading Guide
      • Propositional Logic
        • the logic of propositions.
        • symbols in the language represent conditions in the world, and complex sentences in the language express interrelationships among these conditions.
        • The primary operators are Boolean connectives, such as and, or, and not.
      • Relational Logic
        • Expands upon propositional logic by providing a means for explicitly talking about individual objects and their interrelationships.
        • Expand the language to object constants and relation constants, variables and quantifiers.
      • Herband Logic
        • Provide a means for describing worlds with infinitely many objects.

        • Propositional Language: the set of all propositional sentences that can be formed from propositional vocabulary.
    • Semantics
      • Truth Assignment
      • Truth Value of Compound
      • Definitions
        • A truth assignment satisfies a sentence if and only if the sentence is true under truth assignment
        • A truth assignment falsifies a sentence if and only if the sentence is false under the truth assignment.
        • A truth assignment satisfies a set of sentences if and only if it satisfies every sentence in the set.
        • A truth assignment falsifies a set of sentences if and only if it falsifies at least one sentence in the set.
    • Evaluation
      • Evaluation is the process of determining the truth values of compound sentences given a truth assignment for the truth values of proposition constants.
    • Satisfaction
      • Satisfaction is the opposite of evaluation: we begin with compound sentences and try to figure out which truth assignments satisfy those sentences.
      • A truth table for a propositional language is a table showing all of the possible truth assignments for the proposition constants in the language.
        • For a propositional language with n proposition constants, there are n columns in the truth table and 2^n rows

    • Introduction
    • Logical Properties
      • Categories of sentences
        • Validity
          • A sentence is valid if and only if it is satisfied by every truth assignment.
        • Unsatisfiability
          • A sentence is unsatisfiable if and only if it is not satisfied by any truth assignment.
        • Contigency
          • A sentence is contigent if and only if there is some truth assignment that satisfies it and some truth assignment that falsifies it.
        • Satisfiability
          • A sentence is satisfiable if and only if it is valid or contigent.
        • Falsifiability
          • A sentence is falsifiable if and only if it is unsatisfiable or contingent.
    • Logical Equivalence
      • Equilance
        • We say a sentence a is equivalent to a sentence b if and only if every truth assignment that satisfies a satisfies b and every truth assignment that satifies b satisfies a.
      • Truth Table Method
        • One way of determining whether or not two sentences are logically equivalence is to check the truth table for the proposition constants in the language.
      • Substitutability
        • If a sentence a is logically equivalent to a sentence b, then we can substitute a for b in any Propositional Logic sentence and the result will be logically equivalent to the original sentence.
    • Logical Entailment
      • Entailment
        • A sentence a logically entails a sentence b if and only if every truth assignment that satisfies a also sarisfies b.
        • A set of sentences lgocially entails a sentence b if and only if every truth assignment that satisfies all of the sentence in the set also satisfies b.
    • Logical Consistency
      • Consistency
        • A sentence a is consistent with a sentence b if and only if there is a truth assignment that satisfies both a and b.
        • A sentence a is consistent with a set of sentences if and only if there is a truth assignment that satisfies both the set and a.
    • Properties and Relationships
      • Equivalence Theorem
        • A sentence "a" and a sentence "b" are logically equivalent if and only if the sentence ( a <=> b) is valid.
      • Deduction Theorem
        • A sentence "a" logically entails a sentence "b" if and only if (a => b) is valid.
        • A finite set of sentences { "b", ... "z"} logically entails "a" if and only if the compound sentence (b & c & ... & z => a) is valid
      • Unsatisfiabiliy Theorem
        • A set of sentences ,A, logically entails a sentence "a" if and only if A U (～ a) is unsatisfiable.
      • Consistency Theorem
        • A sentence "a" is logically consistent with a sentence b if and only if the sentence ( a & b) is satisfiable.
        • A sentence "a" is logically consistent with a finite set of sentences {"b", "c", ... "z"} if and only if the compound sentence ( a & b & c & ... & z) is satisfiable.
  • Propositional Proofs
    • Introduction
    • Linear Reasoning
      • Definitions
        • Schema
          • A schema is an expression satisfying the grammatical rules of our language except for the occurence of metavariables in place of various subparts of the expression.
        • Rule of Inference
          • A rule of inference is a pattern of reasoning consisting of some schemas, called premises, and one or more additional schemas, called conclusions.
        • Instance of a Rule of Inference
          • An instance of a rule of inference is the rule obtained by consistently substituting sentences for the metavariables in the rule.
          • The replacement can be an arbitrary expression so long as the result is a legal expression.
          • A rule applies to a set of sentences if and only if there is an instance of the rule in which all of the premises are in the set.
        • Linear Proof
          • A linear proof of a conclusion from a set of premises is a sequence of sentences terminating in the conclusion in which each item is either (1) a premise, (2) an instance of an axiom schema, or (3) the result of applying a rule of inference to earlier items in sequence.
          • Let R be a set of rules of inference. If there exists a proof of a sentence φ from a set Δ of premises using the rules of inference in R, we say that φ is provable from Δ using R. We usually write this as Δ |-R φ

    • Hypothetical Reasoning
      • Definitions
        • Structured Proof
          • Structured proofs have more structures than linear proofs, and in particular, sentences can be grouped into subproofs nested within outer superproofs.
          • We can make assumptions within subproofs; we can prove conclusions from those assumptions; and from those derivations, we can derive implications outside of those subproofs, with our assumption as antecedents and out conclusions as consequents.
        • Implication Intoduction (II)
          • This is achieved by making an assumption and then prove a conclusion. The assumption will be the antecedent, and the conclusion in the subproof will be the consequent.

    • Soundness and Completeness
      • Provability
        • A sentence is provable from a set of premises if and only if there is a finite proof of the conclusion from the premises.
      • Soundness
        • A proof system is sound if and only if every provable conclusion is logically entailed.
        • if Δ ⊢ φ, then Δ ⊨ φ
      • Completeness
        • A proof system is complete if and only if every logical conclusion is provable.
        • if Δ ⊨ φ, then Δ ⊢ φ.
      • The Fitch system is sound and complete for the full language.

    • Clausal Form
      • Definitions
        • Literal
          • A literal is either an atomic sentence or a negation of an atomic sentence.
        • Clausal Sentence
          • A clausal sentence is either a literal or a disjunction of literals.
        • Clause
          • A clause is the set of literals in a clausal sentence.
          • The empty set {} is also a clause, equivalent to an empty disjunction, and therefore is unsatisfiable.
      • Convertion Rules
        • Implications (I)
        • Negation (N)
        • Distribution (D)
        • Operators (O)
    • Resolution Principle
      • Definitions
        • Resolution Derivation
          • A resolution derivation of a conclusion from a set of premises to be a finite sequence of clauses terminating in the conclusion in which each clause is either a premise or the result of applying the Resolution Principle to earlier members of the sequence
          • The resolution is not generatively complete, i.e. it is not possible to find resolution derivations for all clauses that are logically entailed by a set of premise clauses.
  • Relational Logic
    • Introduction
      • Relational Logic
        • Relational logic has two new linguistic features: variables and quatifiers.
    • Syntax
      • Relation Constant
        • Definition
          • Relation constants are used in forming complex expressions by combininh them with an appropriate number of arguments.
        • Arity
          • Each relation constant has an associated arity, the number of arguments with which that relation constant can be combined.
      • Vocabulary
        • A vocabulary consists of a set of object constants, a set of relation constants, and an assignment of arities for each of the relation constants in the vocabulary.
      • Term
        • A term is defined to be a variable or an object constant.
      • Types of Sentences in Relational Logic
        • Relational Sentences
          • The analog of proposition in propositional logic
          • A relational sentence is an expression formed from an n-ary relation constant and n terms.
          • Also called atoms
        • Logic Sentences
          • Analogous to the logical sentences in Propositional Logic
          • There are also negations, conjunctions, disjunctions, implications, and equivalences just like in Pro Log.
        • Quantified Sentencesifier
          • Property
            • Formation
              • Quatified sentences are formed from a quantifier, a variable, and an embedded sentence.
            • Nesting
              • Quantified sentences can be nested within other sentences. ((∀x.(∃y.q(x,y))))
            • Precedence
              • The precedence relations of the logical operators in Relational Logic are the same as in Pro Log, and quantifiers have higher precedence than logical operators.
          • Scope
            • The embedded sentence is called the scope of the quant
          • Universally Quantified Sentence
            • Universally quantified sentence is used to assert that all objects have a certain property.
            • (∀x.(p(x) ⇒ q(x,x)))
          • Existentially Quantified Sentence
            • An existentially quantified sentence is used to assert that some object has a certain property.
            • (∃x.(p(x) ∧ q(x,x)))
          • Ground
            • An expression in Relational Logic is ground if and only if it contains no variables. (like p(a) but not ∀x.p(x))
          • Free vs. Bound
            • An occurence of a variable is free if it is not in the scope of a quantifier of that variable. Otherwise, it is bound.
          • Open vs. Closed
            • A sentence is open if and only if it has free variables. Otherwise, it is closed.
    • Semantics
      • Herbrand Semantics
        • The semantics of Relational Logic presented here is termed Herbrand Semantics
      • Herbrand Base
        • The Herbrand base for a vocabulary is the set of all ground relational sentences that can be formed from the constants of the language.
        • For a set of object constants of size b, there are b^n distinct n-tuples of object constants, and hence there are b^n ground relational sentences for each n-ary relation constant.
      • Truth Assignment
        • A truth assignment for a Relational Logic language is a function that maps each ground relational sentence in the Herbrand base to a truth value.
      • Instance
        • An instance of an expression is an expression in which all free variables have been consistently replaced by ground terms.
        • Consistent replacement here means that if one occurence of a variable is replaced by a ground term, then all occurences of that variable are replaced by the same ground term.
      • Assignment for Universal / Existential sentences
        • A universally quantified sentence is true for a truth assignment if and only if every instance of the scope of the quantified sentence is true for that assignment.
        • An existentially quantified sentence is true for a truth assignment if and only if some instance of the scope of quantified sentence is true for that assignment.

    • Logical Properties
      • Common Quantifier Reversal
        • The Common Quantifier Reversal tells us that reversing quantifiers of the same type has no effect on truth assignment.
        • ∀x.∀y.q(x,y) ⇔ ∀y.∀x.q(x,y)
        • ∃x.∃y.q(x,y) ⇔ ∃y.∃x.q(x,y))
      • Existential Distribution
        • Existential Distribution tells us that it is okay to move an existential quantifier inside of a universal quantifier. (Note that the reverse is not valid)
        • ∃y.∀x.q(x,y) ⇒ ∀x.∃y.q(x,y)
      • Negation Distribution
        • Negation Distribution tells us that it is okay to distribute negation over quantifiers of either type by flipping the quantifier and negating the scope of the quantified sentence.
        • ¬∀x.p(x) ⇔ ∃x.¬p(x)
        • ¬∃x.p(x) ⇔ ∀x.¬p(x)
    • Logical Entailment
      • Examples
        • p(a) ⊨ (p(a) ∨ p(b))
        • p(a) ⊭ (p(a) ∧ p(b))
        • {p(a), p(b)} ⊨ (p(a) ∧ p(b))

      • Transform RL Sentences to PL Sentences
        • Three Steps
          • We first convert our sentences to prenex form, then we ground the result, and we rewrite in PL. Let's look at these steps in turn.
        • Prenex
          • A sentence is in prenex form if and only if it is closed and all of the quantifiers are on the outside of all logical operators.
          • Example
            • From: ∀y.p(x,y) ∨ ∃y.q(y)
            • To: ∀x.∀y.∃z.(p(x,y) ∨ q(z))
